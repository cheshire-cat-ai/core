Cheshire Cat v2 — Plugin DevX

What changed for plugin authors after the giga-refactor: one import surface, no god-object, no dependency-injection ceremony.

openspec: 2026-06-17-giga-refactor-ctx
The one-line summary: the framework used to offer six different ways to reach the same ambient state (the user, the LLM, settings, a hook). They are all gone. Now there is exactly one: from cat import …. The name cat means only the package — there is no self.ccat, no caller, no get_user() dependency to thread around anymore.

1 The big idea

Import names, not objects. Behaviour is resolved per-call.

from cat import llm binds a name to a function. Importing builds nothing. When you call await llm(…), the function resolves the configured provider through the registry at that moment. That is why the friendly import surface can coexist with runtime/plugin/config selection — the name is fixed, the behaviour is late-bound.

Two flavours of ambient access:

KindWhatHow you reach it
Capabilitiesswappable, config-chosen implementations — llm, embedder, memory, auth, hookimported functions, resolved on call
Request contextper-request facts — the current user and the stream callbackimported proxy, backed by contextvars, live per request

user is backed by a contextvars.ContextVar that middleware sets once per request. Each async task gets its own isolated copy, so concurrent requests from alice and bob never see each other's user. Reading user outside a request raises a clear RuntimeError instead of returning stale state.

Why this doesn't bring back the v1 god-object. v1 cycled because cat.x were eagerly-wired attributes referencing each other at construction time. Module-level capability functions are inherently lazy: importing them builds nothing, and a provider that needs another provider just asks at call-time. The registry caches singletons but builds them on first use.

2 Agents

An agent is a verb you run — a fresh instance per run, mutated during the run, thrown away.

Subclass Agent, give it a slug, and override what you need. Run state lives on self: self.task (input), self.result (accumulating output), self.system_prompt, self.tools, and — if you declare an ArgsSchema — self.args. Because every run is a fresh instance, mutating these is safe under concurrency. There is no self.ccat, self.request, or self.factory — ambient needs come from imports.

The simplest agent

from cat import Agent

class StoryBrandAgent(Agent):
    """Your truly StoryBrand Agent."""

    async def get_system_prompt(self):
        return STORY_BRAND_PROMPT

That's it — the base loop() drives the agentic LLM-and-tools cycle for you, streaming to the client.

Typed arguments with ArgsSchema

from pydantic import BaseModel
from cat import Agent
from cat.types import Message, TextContent

class ParrotAgent(Agent):

    slug = "parrot"
    name = "Parrot Agent"
    description = "Just replies back what you just said."

    class ArgsSchema(BaseModel):
        system_prompt: str = "You are a parrot. Repeat what the user says."
        repeat_n: int = 1

    async def get_system_prompt(self):
        return self.args.system_prompt

    async def loop(self):              # override the loop entirely
        for _ in range(self.args.repeat_n):
            self.result.messages.append(
                Message(role="assistant", content=[
                    TextContent(text=self.args.system_prompt + self.task.messages[-1].text)
                ])
            )

Calling a sub-agent — no request threading

Before

result = await self.call_agent(
    "default", task,
    request=self.request,   # 😩
)

After

from cat import call_agent

result = await call_agent("default", task)
self.result.messages.extend(result.messages)

Directives — shaping the agent in three phases

A Directive inspects and changes the agent in place around the loop. It receives the running agent explicitly (it operates on it — so it is intentionally not ambient). The system prompt is reset to its baseline before each step(), which makes per-iteration RAG a non-event. 

from cat.base import Directive

class RecallContext(Directive):

    async def start(self, agent):   # once, before the loop — filter tools, set base prompt
        ...
    async def step(self, agent):    # every iteration, after prompt reset — inject RAG, guardrails
        agent.system_prompt += "\n\nRelevant context: ..."
    async def finish(self, agent):  # once, after the loop — save memories, log
        ...

3 Tools

Just an async function with a docstring. The signature is exactly what the LLM fills in — nothing else.

The framework caller parameter is gone. A tool's input schema (what the LLM sees) now contains only real arguments. Need the current user or a capability? Import it.

Before

@tool
async def send_mail(text, to, caller):
    user = caller.user           # 😩 framework arg
    return f"Sent by {user.name}"

After

from cat import tool, user

@tool
async def send_mail(text: str, to: str):
    """Send email text to an address."""
    return f"Sent mail to {to} by {user.name}"

You can still pass examples for the trigger matching:

from datetime import datetime
from cat import tool

@tool(examples=["what time is it", "get the time"])
async def get_the_time():
    """Useful to get the current time when asked. Input is always None."""
    return f"The current time is {datetime.now()}"

Agents can also carry their own tools as methods, with @tool on a bound method:

from cat import Agent, tool

class Tooler(Agent):
    slug = "tooler"

    @tool
    async def list_modules(self) -> str:
        """List available modules."""
        return [...]

4 Endpoints

Plain FastAPI routes via @endpoint. Reach the user and hooks by import, not by Depends().

get_user() and get_ccat() FastAPI dependencies are removed. Your endpoint signature now contains only its real request parameters; user, config and hook come from cat.

Before

@endpoint.post("/api/v2/uploads")
async def upload_file(
    file = File(...),
    user = get_user(),      # 😩
    ccat = get_ccat(),      # 😩
):
    ...
    await ccat.mad_hatter.execute_hook(
        "after_file_upload", val, caller=ccat)

After

from cat import config, endpoint, user, hook

@endpoint.post("/api/v2/uploads", tags=["Uploads"])
async def upload_file(file: UploadFile = File(...)):
    hashed = uuid5(NAMESPACE_URL, str(user.id))
    save_dir = os.path.join(config.UPLOADS_PATH, str(hashed))
    ...
    await hook("after_file_upload", uploaded)

5 Hooks

One hook name, two jobs: decorate to listen, await to fire.

The trick: arity tells them apart. Firing passes a name and a value; decorating doesn't.

GoalCode
Listen (define)@hook  ·  @hook("custom_name")  ·  @hook(priority=2)
Fire (emit)await hook("after_file_upload", value)
from cat import hook, log

# Listen: receive the value, return the (possibly modified) value.
@hook
async def after_file_upload(uploaded):
    log.info(f"got a file at {uploaded.path}")
    return uploaded

@hook(priority=100)
async def agent_prompt_prefix(prefix):
    return prefix + "\nAlways answer in a friendly tone."
No more cat argument in hooks. The old async def my_hook(value, cat) signature is gone. A hook receives only the value it transforms; everything ambient (user, llm, …) is imported.

Hooks fired by the core that you can listen on include the agent lifecycle ones: before_agent_execution, after_agent_execution, agent_prompt_prefix/agent_prompt_suffix, agent_allowed_tools, plus the slug-scoped variants like before_parrot_agent_execution.

6 LLM & Embedder

Call the configured model in one line. Streaming to the client is automatic.

Before

emb = self.ccat.get_embedder(...)   # 😩
await self.load_settings()           # 😩
vec = emb.embed_query(text)

After

from cat import llm, embedder

reply = await llm("Summarise this in one line.")
vec   = await embedder("text to embed")

Both default to the installation's configured model. Pass an explicit model="provider:model" as a power-user escape hatch:

# Use the installation default (from core settings):
msg = await llm("Hello!")

# Force a specific provider:model — the escape hatch:
msg = await llm("Hello!", model="anthropic:claude-opus-4-8")

# Full control — pass conversation, tools, toggle streaming:
msg = await llm(
    system_prompt="You are concise.",
    messages=task.messages,
    tools=my_tools,
    stream=True,
)

vec = await embedder("text", model="openai:text-embedding-3-small")
Streaming is free. When stream=True, llm() pushes AGUI token events to the current client via the request-context stream callback — no self.request.state to wire up. The slug syntax is "provider:model"; a bare name resolves against the default provider.

7 Writing a provider

LLM/embedder providers are services — subclass ModelProvider, read self.settings, implement llm()/embed().

Providers moved out of core into the llms plugin (OpenAI, Anthropic, Ollama). A provider declares a nested Settings; the registry loads and injects self.settings before setup() runs, so you read config synchronously and never touch the DB. 

from pydantic import BaseModel, Field
from cat import log
from cat.services.model_providers.base import ModelProvider

class Anthropic(ModelProvider):
    """Anthropic Claude models."""

    slug = "anthropic"
    description = "Anthropic Claude models via the Anthropic API."

    class Settings(BaseModel):
        anthropic_key: str = Field(default="", title="Anthropic API Key")

    async def setup(self):                 # self.settings already injected & typed
        from anthropic import AsyncAnthropic
        key = self.settings.anthropic_key
        self.client = AsyncAnthropic(api_key=key) if key else None

    async def list_llms(self):            # powers the dynamic model dropdown
        models = await self.client.models.list()
        return [m.id for m in models.data]

    async def llm(self, model, messages, system_prompt="", tools=[],
                  on_token=None, on_tool_call=None):
        # build request, stream via on_token / on_tool_call, return a Message
        ...

8 Settings

Declare a nested Settings. The registry injects a loaded, typed self.settings before setup().

  • Service.load_settings() / save_settings() are gone. A stateless cat.settings manager owns persistence; self.settings is your per-lifecycle cache.
  • No Settings and no settings_schema() override ⟶ self.settings is None and the service doesn't appear under GET /settings.
  • Loading never destroys data. A corrupted blob falls back to defaults; a changed schema is salvaged field-by-field — every still-valid value is kept, the rest default, nothing is deleted. Give every field a default and migration is a non-event.

Need runtime-derived options (e.g. an enum of available models)? Override the single classmethod — there is one resolution path, no static-vs-dynamic branching:

class MyService(Service):

    class Settings(BaseModel):
        verbose: bool = True

    @classmethod
    async def settings_schema(cls, app):
        # build a dynamic model from runtime data; default returns Settings
        return build_enum_model(await app.get_all("model_providers"))
